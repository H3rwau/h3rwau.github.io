# git

## 1.集中式管理： SVN

有一个主节点master，每个主节点维持着版本历史记录，而从节点跟随主节点即可。

## 2.分布式管理：GIT

每个仓库都维持着版本历史记录，可以各自维护。可以产生各自的分支

分布式的好处：断网可以操作

## 3.使用git的一般流程

sudo apt-get isntall git 安装git客户端

配置姓名和邮箱地址

git config --global user.name 姓名

git config --global user.email 邮箱

文本编辑器配置

git config --global core.editor vim

git config --list 列出git信息



git clone repo地址 文件夹名

git status 展示仓库状态



git pull origin master 拉取



git add  文件名 （最好不要add目录）

git rm 待删除文件

git commit -m "[提交信息]"

git push origin master



git log --all --graph --oneline 显示历史记录




将一个已存在的文件夹转换为一个git项目并托管到github仓库：
1、将命令行终端定位到该文件夹
2、将该文件夹初始化成一个git仓库：\$  git init
3、将该文件夹下的文件（即工作区内容）添加到git缓冲区：\$  git add .
4、将git缓冲区提交到刚刚初始化的git仓库里：\$ git commit -m "本次提交说明"
5、在github上创建一个仓库
6、让本地git仓库关联刚刚创建github仓库：\$ git remote add origin git@server-name:path/repo-name.git
7、将本地git仓库内容同步到远程github仓库上：\$ git push -u origin master






## 4.分支

master 默认分支

HEAD 当前分支

git branch -v 列出所有分支

git branch [分支名] 创建一个分支

git branch -d [分支名] 删除分支



所有文件必须处于未修改状态才可以使用分支状态（分支切换，分支新建，分支合并）

git checkout [分支名] 切换分支

**merge过程可能会产生冲突：**

1. 两个分支修改了同一个文件

   解决冲突方法：找到冲突原因(git status)，查看冲突源文件，修改冲突的位置，使用git add和git commit ，此时提交后会产生合并。

2. 一个分支改了文件，另一个分支删除了文件

   可以选择删除该文件，也可以add该文件



### 不同仓库的同名分支是相对独立的

git remote -v  列出和当前本地仓库相关联的远程仓库

必须是上下游关系才可以Push成功

**如果push失败**

1. git fetch origin master	git fetch：**相当于是从远程获取最新版本到本地，不会自动merge，看不到别人修改的文件**
2. git merge origin/master  **合并，产生了直接下游**
3. git push origin master
