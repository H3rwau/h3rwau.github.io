# c++学习笔记——25.UML统一建模语言，面向对象设计原则

### 1.类与类之间的关系有哪几种？各自的特点是怎样的？

1. 类与类之间的关系有关联，继承(泛化)，聚合，组合，依赖。
2. 关联：关联主要是类的联系，关联可以是单向的也可以是双向的，特点是A has B。
3. 继承(泛化)：继承关系是父子关系，子类在拥有父类的的属性的基础上还可以有其他属性，并且可以使用父类的实例的地方也可以使用子类的实例，特点是A is B。
4. **聚合**：**聚合是整体和部分的关系，而部分和整体又可以分开**，比如电脑是整体，而显示屏，键盘，主板，CPU，GPU，内存，硬盘等都是部分，可以单独存在也可以聚在一起组成电脑。特点是A has B,C,D.
5. **组合**：组合也是整体和部分的关系，**但是部分和整体不可以分开**,比如公司是整体，各个部门是
   部分，这些部门不能脱离于公司存在，这些部门共同组成了公司。特点是A has B,C,D.
6. 依赖：依赖是指这个类的实现需要用到另一个类，这就叫依赖关系。通常情况下需要将另一个类的
   对象作为参数，主要是一个消息传递的过程。

### 2.面向对象设计原则

SOLID原则:

1. **单一职责原则**(Single Responsibility Principle)：**一个类只允许有一个职责**，即只有一个导致该类变更的原因。
2. **开闭原则**（Open Close Principle）:一个软件实体如类、模块和函数应该**对扩展开放，对修改关闭**
3. **里氏替换原则**(Liscov Substitution Principle)：**派生类必须能够替换其基类**。
4. **接口分离原则**（Interface Segregation Principle）:**多个特定的客户端接口要好于一个通用性的总接口**。
5. **依赖倒置原则**（Dependency Inversion Principle）:**依赖抽象，针对接口编程**，而不是依赖实现。**抽象不应该依赖细节**；**细节应该依赖抽象**。高层模块不能依赖低层模块，二者都应该依赖抽象。
