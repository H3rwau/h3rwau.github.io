## 4、请构造合适的例子编写程序，并且回答下面的问题：

#### （a） 如果原来的数据是无符号类型，使用%d打印也是无符号类型吗？

不一定，如果该无符号数大于有符号数表示的范围，那么可能会打印成有符号数。

```c
#include<stdio.h>
int main(void)
{
	int i = 2147483647;/*int 为32位表示数的范围是-2147483648—2147483647*/
	unsigned  int j = 2147483647;/*unsigned  int无符号表示数的范围是 0—4294967295  */

	printf("%d %d %d \n", i, i + 1, i + 2);
	printf("%d %d %d", j, j + 1, j + 2);//j是无符号数，最大可达4294967295,但是用%d打印就只能按照有符号数的规则
	return 0;

}
```

#### （b） 如果原来的数据是有符号类型，使用%u打印也是有符号类型吗？

不一定，因为如果原来的数据是负数，那么打印出来是按照无符号数的规则。

```c
#include<stdio.h>
int main(void)
{
	int i = -2147483647;/*int 为32位表示数的范围是-2147483648—2147483647*/
	unsigned  int j = 2147483647;/*unsigned  int无符号表示数的范围是 0—4294967295  */

	printf("%u %u %u \n", i, i + 1, i + 2);
	printf("%u %u %u", j, j + 1, j + 2);
	return 0;

}
```

## 5、 有符号类型在内存中以二进制补码的形式保存的。

#### （a） 如果给出两个数据的补码形式，如何在不计算10进制数值的情况下进行大小比较？

第一种情况：两个正数相比，正数的补码就是原码，进行相减就行了，如果需要借位，那么证明被减数比较小。

第二种情况：两个负数相比，可以先将补码转换为原码，由于是负数，进行相减后大的那方才是真正小的数。（这样更好理解点）

第三种情况：一正一负相比，看符号位就可以了，正数的符号位是0，而负数的符号位是1，正数肯定是大于负数的。

#### （b） 对于N位的有符号类型，求出最大正数\最大负数\最小负数以及其二进制补码形式。

当N=4时，最大正数的二进制补码是0111，真值为7，最大负数的二进制补码是1111，真值为-1，最小负数的二进制补码是1000，真值为-8.所以当N为4时，补码的表示范围为[-8,7].

可以推出对于N位的有符号类型，补码的表示范围为[-2^(n-1),2^(n-1)-1]

最大正数为2^(n-1)-1,补码形式为0111……111（n-1个1），最大负数为-1，补码形式为n个1，最小负数为-2^(n-1)，补码形式为1000……000（n-1个0）。

## 9、设计程序验证：

#### （a）int类型强制类型转换成short类型，变量（正数、负数）的内存里面会发生什么变化？反过来呢？

int类型强制转换成short类型，变量的内存占的字节会由4变为2.

short类型强制转换成int类型，变量的内存占的字节会由2变为4.

```c
#include<stdio.h>
int main(void)
{
	int i = -5;
	short s = (short)i;
	printf("%d", s);
	return 0;
}
```

#### （b）int类型强制类型转换成float类型，变量（正数、负数）的内存里面会发生什么变化？反过来呢？

```c
#include<stdio.h>
int main(void)
{
	int i = 5;
	int i1;
	float f = (float)i;
	printf("%f ", f);

	i1 = (int)f;
	printf("%d", i1);
	return 0;

}
```

本来i的内存中是 05 00 00 00，通过强制转换成float型的f的内存中为00 00 a0 40,之后再通过i1转换为int型，内存又变化呈了05 00 00 00。
